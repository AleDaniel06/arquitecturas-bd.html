Punto 01 · Definición

¿Qué son las
Arquitecturas
de BD?

Las arquitecturas de base de datos son modelos que definen cómo se estructuran, almacenan y gestionan los datos dentro de un sistema informático. Actúan como el plano fundamental que determina la organización interna de toda la información.

Su correcta elección determina el rendimiento, la seguridad y la capacidad de crecimiento del sistema. Su aplicabilidad depende del entorno: sistemas locales, empresariales o plataformas en la nube.

Punto 02 · Tipos

3 Arquitecturas Principales

Las arquitecturas de base de datos pueden ser centralizadas, cliente-servidor o distribuidas. Cada una responde a necesidades distintas en cuanto a usuarios, volumen de datos y requerimientos de disponibilidad.

🏛️
Tipo 01
Centralizada

Todos los datos residen en un único servidor central. Toda la lógica y almacenamiento están en un solo punto de control.

Fácil administración
Bajo costo inicial
Un solo punto de fallo
Un solo servidor
🖥️
Tipo 02
Cliente-Servidor

Separa la lógica del cliente y del servidor. Múltiples usuarios acceden a través de un servidor central que gestiona peticiones.

Múltiples usuarios
Acceso concurrente
Mayor seguridad
Múltiples clientes
🌐
Tipo 03
Distribuida

Los datos se reparten en varios servidores interconectados. Cada nodo puede operar de forma independiente y en red.

Alta disponibilidad
Tolerancia a fallos
Escalabilidad máxima
Red de servidores
04
Ventajas y Desventajas

Análisis detallado
de cada tipo

🏛️
Centralizada

✦ Ventajas:
Administración simple, bajo costo, fácil mantenimiento, consistencia total de datos y menor complejidad técnica para el equipo.

✦ Desventajas:
Si el servidor falla, todo el sistema cae. No escala con grandes volúmenes. Acceso lento desde ubicaciones remotas.

🖥️
Cliente-Servidor

✦ Ventajas:
Permite múltiples usuarios simultáneos, mayor seguridad mediante control de acceso por roles y mejor separación de responsabilidades.

✦ Desventajas:
Dependencia del servidor central, costo mayor, requiere red estable y administración más compleja que la centralizada.

🌐
Distribuida

✦ Ventajas:
Muy alta disponibilidad, tolerancia total a fallos, escalabilidad horizontal, rendimiento óptimo y acceso global sin latencia.

✦ Desventajas:
Alta complejidad de implementación, costo elevado, garantizar consistencia es difícil y requiere personal especializado.

🎯
¿Cuándo usar cada una?

Centralizada: Aplicaciones personales, pequeñas empresas o proyectos con usuario único.

Cliente-Servidor: Empresas medianas, sistemas universitarios, aplicaciones con varios usuarios en red.

Distribuida: Grandes plataformas digitales, servicios en la nube y sistemas de alcance global.

🔐
Seguridad en cada tipo

Centralizada: Control simple pero con un punto único de vulnerabilidad ante ataques o fallos.

Cliente-Servidor: Autenticación por roles, control de acceso centralizado y auditoría de usuarios.

Distribuida: Cifrado entre nodos, redundancia de datos y auditoría distribuida completa.

💡
Importancia de elegir bien

Una elección incorrecta puede ocasionar pérdida de datos, caídas del sistema o costos excesivos. Analizar el volumen de usuarios, la criticidad del sistema y el presupuesto es fundamental antes de decidir la arquitectura más adecuada para cada proyecto.

Punto 05 · Final

Conclusión &
Recomendación

✦ Conclusión
Elección estratégica

La elección de la arquitectura de base de datos depende directamente del entorno, el volumen de datos y las necesidades específicas del sistema a implementar.

Las arquitecturas modernas priorizan la escalabilidad y disponibilidad.
No existe una solución universal; cada contexto requiere su propio modelo.
La tendencia actual apunta a sistemas híbridos y basados en la nube.
La correcta arquitectura garantiza eficiencia, seguridad y crecimiento sostenible del sistema informático.
→ Recomendación
Para sistemas académicos

Se recomienda la arquitectura cliente-servidor o distribuida, ya que permite múltiples usuarios concurrentes, acceso remoto seguro y mayor protección de la información.

Permite acceso simultáneo de estudiantes y docentes.
Facilita el acceso remoto desde cualquier dispositivo.
Mayor nivel de seguridad para datos académicos sensibles.
Escalable ante el crecimiento de usuarios o datos del sistema.
Estas opciones garantizan rendimiento, concurrencia y protección de los datos académicos a largo plazo.
